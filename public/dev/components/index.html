Job interface · components

Phase 01 verification surface. Each spec card mounts the live custom element built in /components/<name>.js against the design tokens in /css/buhlos.css. No screens ship until these nine pieces are signed off (brief §17).

<job-header>C-01

Sticky top navy band.

Job name · code · % · sync stamp. Used by every job-side screen. No bell, no red dot.

<progress-bar>C-02

Yellow fill on paper-3 track.

Default 6px; small variant 2px (used inside area cards). Tone overrides for green/amber/red.

<area-card>C-03

Dot · name · meta · count · chevron.

Two densities — card (Work Areas list) or row (compact). Snagged areas get a red border.

<task-row> + <seg-status>C-04 · C-05

Hosts seg-status on expand. Hosts snag-button on the inline action.

Snagged is derived — never stored. Tap the row to expand; tap a segment to change status (optimistic).

last event: —
<snag-button>C-06

Red outline. Compact + full-width.

Always carries job/area/task in the event detail. Consumer renders the snag form sheet.

last snag-button:open: —
<product-chip>C-07

Mono code chip.

Hangs off task names. Tap → product spec inline drawer. Brief §20: app-leave for product specs = 0.

last product-chip:open: —
<tools-menu>C-08

Three groups, fixed order. Sheet on phone, rail on desktop.

Run · Capture · Look up. Empty groups never render. Role-aware. Same component in both layouts.

Desktop (rail)
<empty-state>C-09

Plain navy headline, mono caption, optional action.

Never an empty card frame. Every list that can be empty gets one.

<buhl-mark>SYNC

Brand mark + sync indicator.

Subscribes to /components/sync.js. Pulse on upload, both dots stroked when offline.
